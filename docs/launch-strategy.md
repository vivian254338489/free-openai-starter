# Launch Strategy

## Launch Window

Preferred initial publish window:

- Tuesday to Thursday
- UTC afternoon or evening

## Launch Checklist

- push full code, not README-only
- keep upstream credit visible
- include screenshot or architecture image in README later
- include a short demo request in README
- include `TKEN` quickstart example without making the repo look like an ad

## First 100 Star Plan

### GitHub-native

- strong repository title
- strong description
- strong topics
- practical README
- one-click developer quickstart

### External Seeding

- Reddit: `r/programming`, `r/selfhosted`, `r/opensource`, `r/LocalLLaMA` when relevant
- Hacker News: present as a practical gateway tool, not a sales pitch
- Product Hunt: launch only after README and demo look polished

## Messaging Angle

Lead with the pain:

> OpenAI-compatible apps break when one key gets rate limited. This project adds multi-account failover, proxy rotation, and model-aware routing in a lightweight self-hosted gateway.
